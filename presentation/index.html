Week 2: Scalable Patterns

The "Page Object Model" (POM) Implementation.

Strict Separation of Concerns:

  • Pages: Handle Locators & Actions (No assertions).
  • Tests: Handle Logic & Assertions (No locators).
  • Data: Dynamic factories (No hardcoding).

// ContactUsPage.ts (Strict Typing)
export class ContactUsPage {
  readonly submitButton: Locator;
  
  constructor(page: Page) {
    this.submitButton = page.locator('input[type="submit"]');
  }

  // Action only - No Assertion
  async submitForm() {
    await this.submitButton.click();
  }
}

Week 3: Killing Flakiness

Challenge: Random "Google Vignette" Ads (40% Failure Rate).

The AdHandler Pattern

Proactive, low-latency interception of system noise.


// src/utils/AdHandler.ts
static async closeAdOverlays(page: Page) {
  // 1. Scan for known ad selectors
  const selectors = ['#dismiss-button', '.close-video'];
  
  // 2. Conditional Logic (Self-Healing)
  for (const s of selectors) {
    if (await page.isVisible(s)) {
      console.log('üõ°Ô∏è Auto-healing: Closing Ad');
      await page.click(s);
    }
  }
}

Week 4: The Delivery Pipeline

Production-Grade DevOps Implementation.


# .github/workflows/main.yml
jobs:
  test_matrix:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        browser: [chromium, firefox]
    steps:
      - name: Run Tests
        run: npx playwright test
        env:
          # SECURITY: Secrets injected at runtime
          RP_API_KEY: ${{ secrets.RP_API_KEY }}
          CI_ENV: true
      
      - name: Upload Artifacts
        if: failure()
        with:
           path: test-results/
  • ‚úÖ Security: No hardcoded keys. GitHub Secrets used.
  • ‚úÖ Observability: Artifacts only upload on failure (Cost saving).
